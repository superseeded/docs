Validate botanical species names against multiple taxonomic databases with intelligent caching and confidence scoring.

## Overview

This endpoint validates plant species names using a two-stage process:

1. **GBIF Matching**: Fast exact matching against Global Biodiversity Information Facility
2. **TNRS Matching**: Fuzzy matching via Taxonomic Name Resolution Service for unmatched names

Results are categorized by confidence level:
- **Exact matches** (≥90%): High-quality matches ready to use
- **Inexact matches** (50-89%): Partial matches requiring review
- **Invalid matches** (&lt;50%): No valid matches found

## Input Options

This endpoint accepts input in three ways:

### 1. JSON Body with Species Names Array

```bash
curl -X POST https://api.superseeded.ai/v1/validate/species \
  -H "X-API-Key: sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "species_names": [
      "Eucalyptus globulus",
      "Acacia melanoxylon",
      "Banksia integrifolia"
    ]
  }'
```

### 2. JSON Body with File URL (TUS Upload)

For files uploaded via TUS protocol:

```bash
curl -X POST https://api.superseeded.ai/v1/validate/species \
  -H "Authorization: Bearer &lt;delegation_token&gt;" \
  -H "Content-Type: application/json" \
  -d '{
    "file_url": "https://secure.superseeded.ai/files/abc123",
    "filename": "species_list.xlsx",
    "extraction_column": "Botanical Name"
  }'
```

### 3. Direct File Upload (Multipart)

Upload files directly without TUS - reduces round trips for Basic/Pro plans:

```bash
curl -X POST https://api.superseeded.ai/v1/validate/species \
  -H "X-API-Key: sk_..." \
  -F "file=@plant_schedule.xlsx" \
  -F "extraction_column=Species"
```

## Data Sources

### GBIF (Global Biodiversity Information Facility)
- **Coverage**: Global biodiversity database
- **Matching**: Exact canonical name matching
- **Performance**: Fast (typically &lt;100ms)
- **Cache**: 24 hours server-side

### TNRS (Taxonomic Name Resolution Service)
- **Coverage**: World Flora Online (WFO) database
- **Matching**: Fuzzy string matching with confidence scores
- **Performance**: Slower (typically 1-3 seconds per batch)
- **Cache**: 30 days in database

## Performance Tips

1. **Batch Processing**: Send multiple species names in a single request rather than individual requests
2. **Enable Caching**: Keep `cache_results: true` (default) to benefit from cached results
3. **File Processing**: For large datasets, upload files rather than sending large JSON payloads
4. **Column Specification**: Specify `extraction_column` for faster spreadsheet processing

## Use Cases

### Plant Schedule Validation
Perfect for validating botanical names in plant schedules before procurement:

```bash
curl -X POST https://api.superseeded.ai/v1/validate/species \
  -H "X-API-Key: sk_..." \
  -F "file=@landscape_plant_schedule.xlsx" \
  -F "extraction_column=Botanical Name"
```

### Data Quality Assurance
Identify questionable species names in botanical datasets:

- **Exact matches**: Proceed with confidence
- **Inexact matches**: Flag for expert review
- **Invalid matches**: Require correction or removal

### Taxonomic Standardization
Standardize species names across different data sources using authoritative databases (GBIF, World Flora Online).
